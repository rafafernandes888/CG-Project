# Computer Graphics

This repository contains the project developed for the **Computer Graphics** course during the 2025/2026 academic year, using C++ and OpenGL. The goal of this project is to develop a **3D engine**.

## 📁 Project Structure

```text
CG/
├── src/
│   ├── engine/       # Rendering engine source code
│   ├── generator/    # Primitive generator source code
│   ├── models/       # Generated models in .3d format
│   ├── scenes/       # XML scene files
│   ├── shared/       # Shared code between modules
│   └── CMakeLists.txt
├── build/            # Directory where binaries are compiled
├── libs/             # RapidXML library
├── toolkits/         # Additional files and dependencies (GLUT, GLEW)
├── CMakeLists.txt    # Main CMake configuration
└── README.md
```

## 🚀 Build Instructions

### 🐧 Linux / Mac

```sh
$ cmake .
$ make
```

### 🪟 Windows

1. Open the `cmake` terminal.
2. Select the main project folder and the corresponding `build` folder.
3. Configure the project with the appropriate settings. Use `Win32`.
4. Select the `toolkits` folder in the project root.
5. Configure and generate the project.
6. After generating the project, run the following command to build it:

```sh
cmake --build build --config Release
```

This will generate the executables inside the `build/` folder.

## 🖌 Generator

The generator creates model files (`.3d`) containing the vertices of graphical primitives, as well as models generated from Bézier patches.

To generate a primitive model, use the **generator**.

Example for generating a plane:

```sh
# Linux / Mac
$ cd build
$ ./generator plane 2 3 plane_2_3.3d

# Windows
$ cd build\src\generator\Release
$ .\generator.exe plane 2 3 "..\..\..\..\src\models\plane_2_3.3d"
```

**Available primitives and inputs:**

1. `generator plane <length> <divisions> <output>`
2. `generator box <length> <divisions> <output>`
3. `generator sphere <radius> <slices> <stacks> <output>`
4. `generator cone <radius> <height> <slices> <stacks> <output>`
5. `generator bezier <patch_file> <tessellation> <output>`

> Note: All files generated by the **generator** should be created or moved to the `src/models/` folder so that the engine can find them.

## 🌌 Quick Guide: Dynamic Solar System

To visualize the main demo scene (`solar_system.xml`), it is necessary to generate the base models used by the scene.

From the project root directory (`CG`), run the following commands in your terminal. Adjust the path if you are not using Windows.

```sh
# 1. Generate the spheres
$ .\build\src\generator\Release\generator.exe sphere 1 20 20 src\models\sphere_1_20_20.3d
$ .\build\src\generator\Release\generator.exe sphere 1 10 10 src\models\sphere_1_10_10.3d
$ .\build\src\generator\Release\generator.exe sphere 1 8 8 src\models\sphere_1_8_8.3d

# 2. Generate the comet from the Bézier patch
$ .\build\src\generator\Release\generator.exe bezier src\models\teapot.patch 10 src\models\teapot.3d
```

After generating these files, the Solar System scene is ready to be executed in the graphics engine.

## 🎮 Engine

The engine loads XML files containing the scene configuration and renders the generated models using **Vertex Buffer Objects (VBOs)**.

To visualize a scene, run the engine. For example:

```sh
# Linux / Mac
$ cd build
$ ./engine ../src/scenes/solar_system.xml

# Windows
$ cd build\src\engine\Release
$ .\engine.exe ..\..\..\..\src\scenes\solar_system.xml
```

> Note: All available scenes can be found in the `src/scenes/` folder.

## ⌨️ Engine Controls

| Key | Action |
|-----|--------|
| ← / → | Rotate camera horizontally |
| ↑ / ↓ | Rotate camera vertically |
| `i` | Zoom in |
| `o` | Zoom out |
| `a` | Enable/disable axes |
| `m` | Switch rendering mode: wireframe, points or filled |
| `c` | Enable/disable Catmull-Rom curve visualization |
| `r` | Reset camera position |
